# 📅 מערכת סידור עבודה

אתר שבו עובדים מגישים **זמינות חודשית** (מסמנים באילו משמרות הם לא יכולים לעבוד, לכל החודש), והמנהל יוצר סידור עבודה **שבועי** אוטומטי (עם אפשרות עריכה ידנית) ומפרסם אותו. ברירת המחדל בדף העובד היא החודש הבא.

## מבנה השבוע
- 7 ימים (ראשון–שבת)
- 2 משמרות ביום: **יום** 07:00–19:00, **לילה** 19:00–07:00
- 2 עובדים בכל משמרת

## כללים מחייבים (נאכפים בשרת)
- כל עובד יכול לחסום עד **8 ימים בחודש** (יום נספר פעם אחת גם אם נחסמו שתי המשמרות בו).
- **תאריך יעד להגשה (אופציונלי):** המנהל קובע בטאב "הגדרות" עד מתי מגישים זמינות לכל חודש. אחרי התאריך ההגשה ננעלת לעובדים (יום היעד עצמו עדיין פתוח), והמנהל יכול להאריך או להסיר את המגבלה בכל רגע.
- עובד לא עובד יותר מ-**4 משמרות בשבוע**.
- עובד לא עובד **2 משמרות ברצף** — לא שתי משמרות באותו יום, ולא משמרת לילה ומיד אחריה משמרת יום למחרת.

הכללים חלים גם על השיבוץ האוטומטי וגם על עריכה ידנית — סידור שמפר אותם לא יישמר, והעורך מציג התראה חיה.

> 💡 לכיסוי מלא של כל המשמרות (28 שיבוצים בשבוע) צריך לפחות **7 עובדים פעילים** (7 × 4 משמרות = 28). מומלץ 8 ומעלה כדי להשאיר מרווח לאילוצים.

## הרצה מקומית
דרוש Node.js בלבד (אין תלויות חיצוניות):

```
node server.js
```

ואז בדפדפן:
- **דף עובדים:** http://localhost:3000/
- **דף ניהול:** http://localhost:3000/admin — סיסמה ראשונית: `1234` (מומלץ לשנות מיד בטאב "הגדרות")

## איך עובדים עם זה
1. המנהל נכנס לדף הניהול → טאב **עובדים** → מוסיף עובדים. כל עובד מקבל קוד אישי.
2. כל עובד נכנס לאתר מהטלפון/מחשב, בוחר את שמו, מזין את הקוד, ומסמן **על פני חודש שלם** באילו משמרות הוא **לא** יכול לעבוד (עד 8 ימים בחודש).
3. המנהל רואה בטאב **אילוצים** מי הזין ומה, ובטאב **סידור עבודה** לוחץ "⚡ צור סידור אוטומטי".
4. השיבוץ האוטומטי: מכבד את כל האילוצים, מאזן את מספר המשמרות (כולל לילות) בין העובדים, לא משבץ עובד לשתי משמרות באותו יום, ולא משבץ יום מיד אחרי לילה.
5. המנהל מתקן ידנית אם צריך (משבצת עם ⚠️ = ניגוד לאילוץ של העובד) → **שמור** → **פרסם לעובדים**.
6. אחרי הפרסום, כל עובד שנכנס רואה את הסידור, עם המשמרות שלו מודגשות בירוק.

## העלאה לאינטרנט (בחינם, Render)
1. צרו חשבון ב- https://render.com (אפשר עם Google).
2. העלו את התיקייה הזו ל-GitHub (repository חדש).
3. ב-Render: **New → Web Service** → חברו את ה-repo.
4. הגדרות: Runtime = Node, Build Command ריק, Start Command = `node server.js`.
5. מומלץ להוסיף Environment Variable בשם `ADMIN_PASSWORD` עם סיסמת מנהל חזקה.
6. תקבלו כתובת כמו `https://your-app.onrender.com` — שלחו לעובדים.

> ⚠️ **חשוב — שמירת נתונים ב-Render:** בתוכנית החינמית הדיסק נמחק בכל deploy מחדש. כדי שהנתונים יישמרו לתמיד, הוסיפו לשירות **Disk** (בתשלום קטן) עם Mount Path‏ `/opt/render/project/src/data`, או השתמשו ב-Railway/VPS. להרצה מקומית אין בעיה — הכול נשמר בקובץ `data/db.json`.

## איפה הנתונים?
הכול בקובץ אחד: `data/db.json` (עובדים, אילוצים, סידורים). גיבוי = להעתיק את הקובץ.
